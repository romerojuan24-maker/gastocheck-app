COMPONENT STRUCTURE
================================================================================

PAGE: /dashboard/cobracheck/routes

Sección 1: RouteGenerator (Generador de Rutas)
─────────────────────────────────────────────
  Inputs:
    • Dropdown: Cobrador/Comprador (select)
    • Calendario: Fecha (date input)
    • Multi-select: Clientes activos (checkboxes)
    • Botón: "Generar ruta optimizada"

  Output:
    • Tabla: orden, cliente, distancia (km), duración (min), dirección
    • Info: Total distancia + duración estimada
    • Botón: "Enviar al cobrador"

  Backend:
    • Carga clientes de tabla cobra_clients
    • Mock TSP (ordena por nombre + distancias aleatorias)
    • Guarda en tabla cobra_routes
    • Notificación al cobrador (TODO: SMS/push)


Sección 2: MovementsTable (Tabla de Movimientos)
────────────────────────────────────────────────
  Filtros:
    • Cobrador (dropdown)
    • Estado: Pagó / Promesa / No Pagó (select)
    • Rango fechas (desde/hasta)
    • Cliente (multi-select/filtro cliente-side)

  Columnas:
    • Activo (✓)
    • Cliente
    • Tipo (badge: Pagó=verde, Promesa=amarillo, No Pagó=rojo)
    • Monto (right-aligned)
    • Estado (Badge)
    • Motivo/Nota
    • Siguiente fecha (para promesas)

  Features:
    • Auto-refresh cada 5 segundos
    • Colores por tipo movimiento
    • Max 100 registros + scroll
    • Filtrado dinámico


Sección 3: RealtimeMonitor (Monitor en Tiempo Real)
────────────────────────────────────────────────────
  KPIs (refresh cada 3s):
    1. Total Cobrado Hoy (verde) — suma collected_amount
    2. Promesas (amarillo) — count movimientos type='promise'
    3. Movimientos Procesados (azul) — count total

  Progreso Ruta:
    • Barra: "X de Y clientes visitados"
    • Porcentaje completado
    • Indicador visual


Sección 4: ReportsList (Reportes Diarios)
──────────────────────────────────────────
  Tabla:
    • Cobrador
    • Fecha
    • Total Cobrado (verde, suma)
    • Promesas (badge)
    • No Pagó (badge rojo)
    • Movimientos (count)
    • Estado (Pendiente/Aprobado/Rechazado)
    • Acciones: ✓ Aprobar | ✗ Rechazar

  Features:
    • Genera automáticamente de movimientos
    • Agrupa por user_id + date
    • Botones deshabilitados si status ≠ pending
    • Max 20 reportes


CUSTOM HOOKS
================================================================================

1. useCompanies(companyId)
   ├─ Returns: CompanyMember[]
   └─ Fields: id, email, fullName, role ('collector'|'operator')

2. useClients(companyId, collectorId?)
   ├─ Returns: CobraClient[]
   └─ Filtra: status='active', order by name

3. useOptimizeRoute(clientIds, clients)
   ├─ Returns: { route, loading, error, optimize }
   └─ route: OptimizedRoutePoint[] (order, clientId, distance, duration)

4. useMovementsList(companyId, filters)
   ├─ Returns: { movements, loading, error, refetch }
   └─ filters: { collectorId, clientId, status, dateFrom, dateTo }

5. useDailyReports(companyId, filters)
   ├─ Returns: { reports, loading, error, refetch, approve, reject }
   └─ reports: DailyReportRow[] (agrupado por user + date)

DATA FLOW
================================================================================

Flujo 1: Generar Ruta
─────────────────────
Supervisor selecciona cobrador → Selecciona clientes → Click generar
   ↓
useOptimizeRoute() → Mock TSP (o backend real)
   ↓
Tabla con orden optimizada aparece
   ↓
Click "Enviar cobrador" → INSERT cobra_routes → SMS/push


Flujo 2: Monitorear en Vivo
────────────────────────────
MovementsTable.load() cada 5s
   ↓
Cobrador registra movimiento (mobile app)
   ↓
Movimiento aparece en tabla (verde/amarillo/rojo)
   ↓
RealtimeMonitor actualiza KPIs


Flujo 3: Aprobar Reportes
──────────────────────────
Fin de día → ReportsList agrupa movimientos
   ↓
Muestra totales por cobrador
   ↓
Supervisor: ✓ Aprobar (status → 'approved')
   ↓
Reporte bloqueado


DATABASE REQUIREMENTS
================================================================================

Existing Tables:
  ✓ cobra_clients (CobraClient)
  ✓ cobra_movements (CobraMovement)
  ✓ company_members + profiles (SessionUser, Collectors)

New Tables Required:
  □ cobra_routes
    - id (UUID PK)
    - company_id (FK)
    - collector_id (FK profiles)
    - date (DATE)
    - clients_order (TEXT[] — array de client IDs)
    - status ('pending'|'in_progress'|'completed')
    - total_distance (FLOAT)
    - estimated_duration (INT)
    - created_at, updated_at (TIMESTAMP)
    - Indexes: (company_id, date), (collector_id)

  □ cobra_daily_reports (optional, si quieren persistencia)
    - id (UUID PK)
    - company_id (FK)
    - collector_id (FK)
    - date (DATE)
    - total_collected (NUMERIC)
    - promise_count, not_paid_count, movements_count (INT)
    - status ('pending'|'approved'|'rejected')
    - created_at, updated_at (TIMESTAMP)
    - Unique: (company_id, collector_id, date)
